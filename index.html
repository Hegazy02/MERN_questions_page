📝 أسئلة HTML & CSS

إيه الفرق بين em, rem, و px؟ وامتى تستخدم كل واحد؟

px: وحدة مطلقة ثابتة، كل بكسل = 1/96 بوصة. بنستخدمها لـ borders والحاجات اللي عايزينها تبقى ثابتة.

em: وحدة نسبية تعتمد على font-size للـ parent element. لو الأب font-size بتاعه 16px و حطيت 2em يبقى الناتج 32px.

rem: وحدة نسبية تعتمد على font-size للـ root element (html). أفضل للـ responsive design عشان كله مرتبط بحاجة واحدة.

متى نستخدم مين: px للـ borders، rem للـ font-sizes والـ spacing، em للـ components اللي عايزة تكبر وتصغر مع المحتوى.

إيه الفرق بين absolute, relative, fixed, و sticky؟

relative: الـ element يفضل في مكانه الطبيعي بس يقدر يتحرك بـ top/left/right/bottom بدون ما يأثر على العناصر التانية.

absolute: الـ element يطلع من الـ document flow ويتموضع بالنسبة لأول parent ليه positioned (مش static).

fixed: زي absolute بس يتموضع بالنسبة للـ viewport، يفضل في مكانه حتى لو scroll.

sticky: مزيج من relative و fixed، يفضل relative لحد ما يوصل لنقطة معينة في الـ scroll يبقى fixed.

يعني إيه Critical Rendering Path وإزاي بيأثر على سرعة الموقع؟

Critical Rendering Path هي الخطوات اللي بيعملها البراوزر عشان يعرض الصفحة:

1. HTML Parsing: يقرا الـ HTML ويعمل DOM tree

2. CSS Parsing: يقرا الـ CSS ويعمل CSSOM tree

3. Render Tree: يدمج الـ DOM والـ CSSOM

4. Layout: يحسب مواضع وأحجام العناصر

5. Paint: يرسم العناصر على الشاشة

التحسين: قلل الـ CSS والـ JS، استخدم resource hints زي preload، و inline الـ critical CSS.

إيه الفرق بين inline, block, و inline-block؟

inline: زي <span> و <a>، يقعد جنب بعض في نفس السطر، مايقبلش width/height، بس يقبل padding/margin أفقي بس.

block: زي <div> و <p>، كل element ياخد السطر كله، يقبل width/height/padding/margin عادي.

inline-block: أحسن ما في الاتنين، يقعد جنب بعض زي inline بس يقبل width/height زي block.

استخدامات: inline للـ text styling، block للـ layout، inline-block للـ buttons والـ navigation items.

إيه المقصود بـ Paint & Layout في الـ Rendering Engine؟ وإزاي تقلل تأثيرهم على الأداء؟

Layout (Reflow): لما البراوزر يحسب مواضع وأحجام العناصر تاني. ده بيحصل لما تغير width/height/position.

Paint (Repaint): لما البراوزر يرسم العناصر تاني بدون ما يغير مواضعها. ده بيحصل لما تغير color/background.

تقليل التأثير:

• استخدم transform و opacity بدل من تغيير الـ position

• استخدم will-change للعناصر اللي هتتحرك

• جمّع الـ DOM changes في batch واحد

• استخدم requestAnimationFrame() للـ animations

إزاي تتجنب مشكلة Cumulative Layout Shift (CLS) وتخلي تجربة المستخدم أحسن؟

CLS هو لما العناصر تغير مكانها فجأة وهي بتتحمل، وده بيضايق المستخدم جداً.

أسباب CLS: الصور بدون dimensions، الخطوط بتتحمل، الإعلانات بتظهر فجأة.

الحلول:

• حط width و height للصور دايماً

• استخدم font-display: swap للخطوط

• احجز مكان للإعلانات مسبقاً

• استخدم aspect-ratio للعناصر المتغيرة

• تجنب إضافة محتوى فوق المحتوى الموجود

إيه الفرق بين z-index والـ stacking context؟

z-index: رقم بيحدد مين قدام مين في الـ z-axis، العنصر بالرقم الأكبر يظهر قدام.

Stacking Context: منطقة منفصلة للترتيب، زي "طبقة" منفصلة عن باقي العناصر.

إزاي يتكون Stacking Context:

• الـ root element (html)

• positioned elements مع z-index مش auto

• elements مع opacity أقل من 1

• elements مع transform/filter/perspective

مهم: z-index يشتغل بس جوا نفس الـ stacking context، مايقدرش يعدي العناصر في context تاني.

تعمل إيه عشان تتأكد إن موقعك متجاوب مع كل الشاشات؟

الأساسيات:

• استخدم meta viewport في الـ head

• فكر Mobile-First (ابدأ بالموبايل وطلع لفوق)

• استخدم CSS Grid و Flexbox للـ layouts

Media Queries: حط breakpoints للشاشات المختلفة

• Mobile: 320px - 768px

• Tablet: 768px - 1024px

• Desktop: 1024px+

الوحدات المرنة: استخدم %, vw, vh, rem بدل من px

الاختبار: جرب على أجهزة حقيقية، مش developer tools بس

⚡ أسئلة JavaScript

إيه الفرق بين null و undefined؟

undefined: لما متغير اتعرّف بس ماتديلهوش قيمة، أو لما function مترجعش حاجة.

let x; // x is undefined

null: قيمة بتحطها بنفسك عشان تقول إن المتغير فاضي عمداً.

let x = null; // x is intentionally empty

الفروق:

• typeof undefined يطلع "undefined"

• typeof null يطلع "object" (دي bug قديمة في JavaScript)

• null == undefined يطلع true

• null === undefined يطلع false

إزاي بيشتغل الـ Event Loop جوا المتصفح؟ وليه مهم تفهمه كويس؟

Event Loop هو اللي بيدير تنفيذ الكود الـ Asynchronous في JavaScript.

المكونات:

• Call Stack: مكان تنفيذ الكود الـ synchronous

• Task Queue: طابور الـ callbacks (setTimeout, DOM events)

• Microtask Queue: طابور الـ Promises (أولوية أعلى)

الآلية: Event Loop يشوف الـ Call Stack فاضي ولا لأ، لو فاضي ياخد من Microtask Queue الأول، بعدين Task Queue.

ليه مهم: عشان تفهم ترتيب تنفيذ الكود وتتجنب blocking الـ UI thread.

إيه الفرق بين let, const, و var من حيث الـ Scope والـ Hoisting؟

Scope:

• var: Function-scoped أو Global-scoped

• let/const: Block-scoped (جوا {} بس)

Hoisting:

• var: يتحرك لأول الـ function ويبقى undefined

• let/const: يتحرك بس يدخل في Temporal Dead Zone

Re-declaration:

• var: ممكن تعرفه تاني في نفس الـ scope

• let/const: مايتعرفش تاني

Assignment:

• const: لازم تديله قيمة من الأول ومتقدرش تغيرها

الفرق بين == و ===؟

== (Loose Equality): بيقارن القيم بس، ويعمل type conversion لو احتاج.

'5' == 5 // true (string بقى number)

true == 1 // true (boolean بقى number)

=== (Strict Equality): بيقارن القيمة والـ type مع بعض.

'5' === 5 // false (string مش زي number)

true === 1 // false (boolean مش زي number)

النصيحة: استخدم === دايماً عشان تتجنب المفاجآت الغريبة في المقارنات.

استثناءات: بس لو عايز تشوف حاجة null أو undefined ممكن تستخدم x == null

تستخدم call, apply, و bind إمتى؟ وإزاي؟

كلهم بيغيروا الـ this context في الـ function.

call: ينفذ الـ function فوراً مع this جديد

func.call(newThis, arg1, arg2)

apply: زي call بس الـ arguments في array

func.apply(newThis, [arg1, arg2])

bind: يرجع function جديد مع this محدد (مش ينفذه)

const newFunc = func.bind(newThis)

متى نستخدم مين:

• call/apply: لما عايز أنفذ فوراً

• bind: لما عايز أحفظ function مع context معين لاستخدام لاحق

يعني إيه Closure؟ وإزاي تشرحها لحد مبتدئ؟

Closure: لما function تقدر توصل للمتغيرات اللي في الـ scope الخارجي حتى بعد ما الـ outer function تنتهي.

مثال بسيط:

function outer(x) {

  return function inner(y) {

    return x + y; // inner شايف x

  };

}

الشرح للمبتدئ: فكر فيها كده، الـ inner function "فاكرة" البيئة اللي اتولدت فيها، زي واحد سافر بلد تانية بس لسه فاكر بيته الأصلي.

الاستخدامات: Data privacy، Module pattern، Event handlers
